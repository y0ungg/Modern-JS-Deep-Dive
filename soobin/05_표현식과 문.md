# ☑️ 5. 표현식과 문

✍️ 2022.07.12(Tues)

## 📎 5.1\_값

> 🧐 <i><strong>값(value)</strong></i>이란? <br/>
> : 표현식(expression)이 평가(식을 해석해서 값을 생성하거나 참조하는 것)되어 생성된 결과

- 모든 값은 데이터 타입을 가지며, 메모리에 2진수, 즉 bit의 나열로 저장됨.

```js
// 변수 sum에는 10+20이 할당되는 것이 아니라 10+20이 평가된 결과인 숫자 값 30이 할당됨.
var sum = 10 + 20;
```

<br/>

## 📎 5.2\_리터럴

> 🧐 <i><strong>리터럴(literal)</strong></i>이란? <br/>
> : 사람이 이해할 수 있는 문자를 사용하여 값을 생성하는 표기법

- JS엔진은 코드가 실행되는 시점인 런타임에 리터럴을 평가해 값을 생성 <br/>
  👉 즉, 리터럴은 값을 생성하기 위해 미리 약속한 표기법

<br/>

## 📎 5.3\_표현식

> 🧐 <i><strong>표현식(expression)</strong></i>이란? <br/>
> : 값으로 평가될 수 있는 문(statement) <br/>
> 👉 즉, 표현식이 평가되면 새로운 값을 생성하거나 기존 값을 참조

✋ 리터럴은 값으로 평가 → 따라서 리터럴도 표현식!

- 표현식은 리터럴, 식별자(변수, 함수 등의 이름), 연산자, 함수 호출 등의 조합으로 이뤄질 수 있다. <br/>
  👉즉, 값으로 평가될 수 있는 문은 모두 표현식!!

* 표현식과 표현식이 평가된 값은 동등한 관계, 즉 `동치(equivalent)`다.

<br/>

## 📎 5.4\_문

> 🧐 <i><strong>문(statement)</strong></i>이란? <br/>
> : 프로그램을 구성하는 기본 단위이자 최소 실행 단위 <br/><br/>
> ➰ 프로그램: 문의 집합으로 이뤄진 것 <br/>
> ➰ 프로그래밍: 문을 작성하고 순서에 맞게 나열하는 것

- 문은 여러 토큰으로 구성

> 🧐 <i><strong>토큰(token)</strong></i>이란? <br/>
> : 문법적인 의미를 가지며, 문법적으로 더 이상 나눌 수 없는 코드의 기본 요소를 의미 <br/>
> ex) 키워드, 식별자, 연산자, 리터럴, 특수 기호 ...

- 문을 명령문이라고도 부름 👉 컴퓨터에 내리는 명령

<br/>

## 📎 5.5\_세미콜론과 세미콜론 자동 삽입 기능

- 세미콜론(;): 문의 종료

- 즉, JS엔진은 세미콜론으로 문이 종료한 위치를 파악하고 순차적으로 하나씩 문을 실행 <br/>
  👉 문을 끝낼 때는 세미콜론을 붙여야 함!

> ✋ 단, 문을 중괄호로 묶은 코드 블록(`{...}`) 뒤에는 세미콜론을 붙이지 않음 <br/>
> 🤷‍♀️ why? 문의 종료를 의미하는 자체 종결성을 갖기 때문

> ✋ 여기서 문의 끝에 붙이는 세미콜론은 옵션! 👉 즉, 세미콜론은 생략 가능 <br/>
> 🤷‍♀️ why? JS엔진이 소스코드 해석시 문의 끝이라고 예측되는 지점에 '세미콜론 자동 삽입 기능'이 암묵적으로 수행되기 때문

<br/>

## 📎 5.6\_표현식인 문 & 표현식이 아닌 문

➰ 표현식인 문: 값으로 평가될 수 있는 문 (ex. 할당문) <br/>
➰ 표현식이 아닌 문: 값으로 평가될 수 없는 문 (ex. 변수 선언문)
